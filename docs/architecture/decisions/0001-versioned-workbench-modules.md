# ADR-0001：采用版本化本地工作台模块注册表

## 状态

Accepted — 2026-07-10

## 背景

当前应用已经具备无损 C 投影、结构选择、受控编辑和本地运行能力，但扩展入口仍集中在渲染器主文件与固定检查器标签中：

- `src/main.ts` 已达到 746 行、37 个顶层函数，同时协调导入、解析、选择、解释、编辑和运行。
- `src/ui/workbench-shell.ts` 写死“解释 / 编辑 / 运行”三个检查器视图。
- 未来算法元素、算法设计工具、比较器和改进流程若继续直接加入主文件，会增加回归范围和跨功能耦合。
- 源码必须继续是唯一事实源；扩展模块不能绕过补丁规划、重解析和确认门禁直接改写 C 文本。

本项目是单机 Electron 应用，不需要网络插件市场、第三方代码沙盒或微服务。这里的“可扩展”应表示稳定的本地模块边界，而不是运行任意外部代码。

## 决策

采用一个小型、版本化、类型安全的本地工作台模块注册表：

- 每个模块提供稳定的 `id`、`version`、标签和能力声明。
- 模块可贡献检查器视图、命令和版本化算法元素定义。
- 注册表负责全局 ID 冲突检查、稳定排序、能力查询和不可变快照。
- UI 外壳只根据注册表快照生成入口，不再知道所有具体功能。
- `src/workbench/` 的项目内依赖采用白名单：只能指向自身或明确的 `src/shared/` 纯共享契约；不得导入 `src/core/editing/`、UI、app、renderer 或 Electron。
- 算法元素定义只声明类型、版本和类别；算法图、课程语义和持久化模型将在获得 M5/M6 分析事实后单独决策，避免现在锁死错误抽象。
- 所有源码变更仍必须进入现有结构化编辑引擎；模块注册不构成新的写入通道。
- 第一阶段只加载随应用编译的本地模块，不支持动态下载或执行第三方代码。

## 备选方案

### 方案 A：继续在主文件中硬编码

- 优点：短期代码最少，容易追踪调用。
- 缺点：每个新功能都修改同一个协调器和外壳，测试范围持续扩大，算法元素无法独立演进。
- 放弃原因：已经出现单文件职责过载，继续累加与长期目标冲突。

### 方案 B：立即建设通用第三方插件系统

- 优点：理论上扩展能力最大，可由外部开发者发布功能。
- 缺点：需要权限、签名、迁移、兼容性和恶意代码隔离；会扩大 Electron 安全边界。
- 放弃原因：当前没有第三方加载需求，成本和风险远高于价值。

### 方案 C：现在定义完整算法节点图模型

- 优点：能提前统一算法编辑、连接和持久化接口。
- 缺点：M5/M6 的程序分析与算法识别事实尚未落地，过早建模会把推测固化为公共契约。
- 放弃原因：先稳定扩展槽位，等确定性分析模型成熟后再定义算法文档。

## 后果

### 正面

- 新增检查器或算法元素时不需要修改外壳的分支逻辑。
- 功能冲突在注册阶段失败，而不是在运行时静默覆盖。
- 算法元素可以按定义版本迁移，支持后续创建、比较和改进算法。
- 保留模块化单体部署，不引入网络服务或额外进程。

### 负面

- 模块清单和贡献契约增加了一层显式结构。
- 契约变更需要版本迁移与兼容性测试。
- 当前阶段只有内置模块，不能把“注册表”宣传为第三方插件平台。

### 中性

- 现有解释、编辑和运行能力成为第一批内置模块，行为不变。
- 视觉主题与模块契约相互独立，后续可以更换外观而不改变算法能力。

## 实施与迁移

1. 增加纯 TypeScript 的模块契约、注册表和冲突测试。
2. 让检查器标签从注册表快照生成，并保持现有 DOM 行为和键盘导航。
3. 把目标选择、解释渲染等职责从 `src/main.ts` 提取为独立模块。
4. 用编译后依赖图与 TypeScript 源码导入门禁共同禁止工作台契约导入 DOM、Electron 或分析层写接口。
5. M5/M6 后另建 ADR，定义算法文档、元素关系与版本迁移。

回滚时可恢复固定的三个检查器描述；注册表不持有项目数据，因此回滚不会转换或丢失源码。

## 架构适应度检查

- 注册表定向测试：重复模块、视图、命令和元素 ID 必须拒绝。
- 同一组模块无论输入顺序如何，快照排序必须确定且深冻结。
- `dependency-cruiser` 检查编译后的运行时图：`src/workbench/` 的项目内依赖只能指向自身或 `src/shared/`。
- TypeScript 源码导入测试在编译擦除前检查同一白名单及 `src/core/` → `src/workbench/` 禁令；负例必须证明 `import type` / `export type` 违规会失败。当前 TypeScript 7 超出 dependency-cruiser 17.4.3 的 TypeScript 解析支持范围，因此不能用编译后巡检冒充类型依赖门禁。
- 循环依赖保持为零。
- 自动检查 `src/main.ts` 不超过 500 行；新增算法能力不得重新把具体 UI 分支写回主文件。
