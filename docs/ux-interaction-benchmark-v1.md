# UX interaction benchmark v1

这份基准把“更顺手”拆成可重复验证的交互预算。它验证关键路径是否退化，不把自动测试通过率冒充产品成熟度评分。

## 复现

```bash
npx vitest run tests/ux/ux-interaction-benchmark.test.ts
```

门禁与手工任务的机器可读定义在
[`benchmarks/ux-interaction-v1.json`](../benchmarks/ux-interaction-v1.json)。修改阈值必须同时说明测量单位；不允许在代码中写死一个“产品总分”。

## 自动门禁

| 关键路径             |                        阈值 | 当前可核验证据                                             |
| -------------------- | --------------------------: | ---------------------------------------------------------- |
| 无输入程序首次运行   |               最多 1 次激活 | 初始主操作为运行；工作区只有一个 `data-primary-action`     |
| stdin 程序首次运行   | 最多 2 次激活，不计文本键入 | 第一次打开按需输入；第二次提交并运行                       |
| 定位下一个确定性问题 |               最多 1 次激活 | `F8` 下一项，`Shift+F8` 上一项，并公开 `aria-keyshortcuts` |
| 工作区常驻主操作     |                   恰好 1 个 | 运行保持唯一主操作；观察与分析使用固定位置的次级入口       |
| 教学轨常驻主操作     |                   最多 1 个 | 补全、故障注入、下一任务互斥显示                           |
| 教学轨初始信息区     |                   最多 4 区 | 任务头、证据状态、主操作、折叠详情                         |
| 高级运行区           |               默认展开 0 个 | 输出与诊断默认折叠                                         |

动作数只统计一次鼠标、触控板或键盘命令；连续文本输入不按字符计数。范围从“工作区已打开且源码投影就绪”开始，启动应用和选择文件不混入算法工作区的操作预算。

## 手工任务记录

自动门禁无法证明用户真的能发现入口。每次要声称成熟度提升时，至少用同一构建重复以下三项，并记录 `构建 SHA / 设备 / 完成秒数 / 误触次数 / 求助次数 / 是否完成`：

1. 打开无输入 C 程序并得到首次真实输出：目标 10 秒，0 误触，0 求助。
2. 为 `scanf` 程序输入一组数据并得到首次真实输出：目标 30 秒，最多 1 次误触，0 求助。
3. 依次定位三个确定性问题并回到第一个：目标 15 秒，0 误触，0 求助。

这些目标是本项目的内部交互预算，不是行业统计。单人自测只能发现回归；若要把结果用于对外成熟度声明，应至少让未参与开发的目标用户执行任务，并同时报告失败样本。

## 成功案例元素

- [VS Code 默认快捷键](https://code.visualstudio.com/shortcuts/keyboard-shortcuts-windows.pdf)用 `F8` / `Shift+F8` 导航下一个和上一个问题；本项目沿用这个已形成用户记忆的模式。
- [Node-RED 节点工作区](https://nodered.org/docs/user-guide/editor/workspace/nodes)把状态放在节点附近，并支持在连线上快速插入节点；本项目用同样的“结果靠近对象、减少往返”原则，但继续受 C 解析与 CFG 安全门约束。
- [JetBrains Features Trainer](https://www.jetbrains.com/help/idea/feature-trainer.html)以本地学习项目和真实任务完成状态推进课程；本项目的教学轨同样依赖真实运行与 Trace 证据，而不是连续点击下一步。

## 这份基准不测什么

视觉层级、连线手感、首次发现率、长任务疲劳、无障碍完整性和真实学习效果仍需实机观察或用户测试。自动门禁全部通过，只能说明这里列出的交互合同成立，不能单独证明“达到 7.5”或“等同成熟产品”。
