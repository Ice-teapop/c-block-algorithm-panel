# AlgoLatch v0.0.2

`v0.0.2` introduces the **AlgoLatch** product name and the first fail-closed
signed macOS release path. It does not replace or mutate the historical
`v0.0.1` tag and unsigned artifacts.

## Install

1. Download `AlgoLatch-0.0.2-universal.dmg` from this Release.
2. Open the DMG and drag **AlgoLatch** into **Applications**.
3. Open AlgoLatch from Applications.

The attached formal DMG is uploaded only after Developer ID signing, Hardened
Runtime, Apple notarization, stapling, quarantine-aware Gatekeeper assessment,
Universal binary checks, and an installed-application regression all pass. No
terminal command, Control-click bypass, or Gatekeeper disablement is required.

`SHA256SUMS.txt` is included for users who want an independent integrity check.

## What changed

- Renamed the user-facing application, window, `.app`, DMG, About information,
  and release metadata to **AlgoLatch**.
- Kept the existing bundle identifier, internal package name, local preference
  keys, managed Documents workspace, repository address, and project data
  locations compatible with `v0.0.1`.
- Added an Apple Developer ID release configuration with a fixed one-entitlement
  allowlist for Electron 43.
- Separated unsigned development builds into `release-beta/` with `unsigned` in
  the filename so they cannot be confused with a formal download.
- Made published Release assets immutable and required the release tag commit to
  be reachable from `main`.
- Added live version, platform, Electron, license, and project links to
  Settings → About.
- Rewrote the product introduction and added a current architecture overview.

## Existing installations

AlgoLatch reads the same managed projects and local settings as the earlier
application. Because the `.app` filename changed, macOS may leave both apps in
Applications. After AlgoLatch opens successfully and shows the expected
projects, the old `C 积木算法面板.app` can be removed manually. Neither the new
app nor the release workflow deletes user projects or settings.

## Distribution boundary

The public `v0.0.1` DMG remains unsigned and unnotarized. The separate unsigned
Beta builder is for local development only and is never attached to this
Release.
