# AlgoLatch v0.0.2

`v0.0.2` introduces the **AlgoLatch** product name and an independent,
fail-closed release path for a signed Windows 10/11 x64 package. It does not
replace or mutate the historical `v0.0.1` tag and unsigned artifacts.

The active `v0.0.2` release workflow produces only a verified Windows
installer. macOS remains on an unsigned local test channel until a future
version can use Developer ID signing and Apple notarization.

## Install

### Windows

1. Download `AlgoLatch-Setup-0.0.2-x64.exe` and `SHA256SUMS.txt` from the same
   Release.
2. Run the one-click per-user installer. Administrator access is not required.
3. Open AlgoLatch from the Start menu. The locked C toolchain is bundled; no
   Visual Studio, LLVM installation, or `PATH` change is required.

The formal Windows installer is uploaded only after Authenticode signing,
signature verification, installation, launch, project creation, native C
compile/run, uninstall, and project-retention checks all pass. Authenticode
proves publisher identity and file integrity, but a new certificate or a low
download count can still trigger Microsoft SmartScreen reputation warnings.

`SHA256SUMS.txt` covers the formal Windows installer for users who want an
independent integrity check. Windows publishing does not require Developer ID,
Apple notarization, or a macOS artifact.

### macOS test channel

`v0.0.2` does not promise or attach a macOS DMG. Maintainers can build the
explicitly unsigned local test package with `npm run dist:mac:beta`. It is not
Apple-verified or a stable release asset.

After copying an unsigned test build into **Applications**, Gatekeeper can
require Control-click → Open or explicit approval in **System Settings →
Privacy & Security**. Do not disable Gatekeeper globally. A future formal macOS
version will require Developer ID signing, Hardened Runtime, Apple notarization,
stapling, quarantine-aware Gatekeeper assessment, Universal binary checks, and
an installed-application regression.

## What changed

- Renamed the user-facing application, window, `.app`, DMG, About information,
  and release metadata to **AlgoLatch**.
- Moved the canonical GitHub repository to `Ice-teapop/algolatch`; GitHub keeps
  redirecting the earlier repository URL.
- Kept the existing bundle identifier, internal package name, local preference
  keys, managed Documents workspace, and project data locations compatible
  with `v0.0.1`.
- Added an Apple Developer ID release configuration with a fixed one-entitlement
  allowlist for Electron 43.
- Separated unsigned development builds into `release-beta/` with `unsigned` in
  the filename so they cannot be confused with a formal download.
- Made published Release assets immutable and required the release tag commit to
  be reachable from `main`.
- Added live version, platform, Electron, license, and project links to
  Settings → About.
- Rewrote the product introduction and added a current architecture overview.
- Added Windows 10/11 x64 execution with a bundled, digest-locked llvm-mingw
  toolchain and a Job Object broker that bounds process count, memory, CPU, and
  process-tree lifetime.
- Added a one-click, per-user NSIS installer with no administrator or external
  compiler requirement. Uninstalling the app preserves managed projects in
  Documents.
- Added Windows `windows-2025` CI and an independent Windows release gate. An
  unsigned Preview has passed build, install, launch, project creation, native
  C compile/run, uninstall, and project-retention validation, but is not a
  stable release asset.
- Kept signed and unsigned macOS package configurations available for future
  use and local testing, while excluding macOS artifacts from the active
  `v0.0.2` GitHub release workflow.

## Existing installations

AlgoLatch reads the same managed projects and local settings as the earlier
application. Because the `.app` filename changed, macOS may leave both apps in
Applications. After AlgoLatch opens successfully and shows the expected
projects, the old `C 积木算法面板.app` can be removed manually. Neither the new
app nor the release workflow deletes user projects or settings.

Windows has no earlier stable AlgoLatch installation to migrate. Its managed
workspace uses the same `C Algorithm Workbench` structure under the user's
Documents folder.

## Distribution boundary

The public `v0.0.1` DMG remains unsigned and unnotarized. Separate unsigned
macOS and Windows Beta builders are for development and installed-state
validation only; they are never attached to this Release. The formal `v0.0.2`
Windows asset is not published until its Authenticode credentials and Windows
installed-state gates pass. It does not wait for Apple credentials. A signed
macOS package can be considered only in a future version after all macOS gates
pass.
