# C 积木算法面板 v0.0.1

发布日期：2026 年 7 月 14 日

`v0.0.1` 是 C 积木算法面板的首个公开版本，也是重置后公开版本线的起点。
它将作为普通 GitHub Release 发布，而不是 prerelease。

此前的 `v0.1.0-beta.1` 至 `v0.1.0-beta.12` 是开发阶段快照。它们记录了
项目的演进过程，但不代表 `v0.0.1` 是面向用户的版本降级。

## 这个版本是什么

C 积木算法面板是一款面向本科 C、数据结构与算法学习的本地 macOS
工作台。它始终把 `main.c` 作为唯一可执行事实源，并通过紧凑积木、经过
验证的控制连线、源码、真实运行路径和分析证据呈现同一个程序。

你可以导入已有 C 代码，在安全支持的范围内组装结构，随时查看真实源码，
在本机运行程序，并用 Trace、诊断和可比较数据调试或改进算法。

## 重点功能

### 在不丢失 C 源码的前提下搭建算法

- 保真导入任意单文件 C17，保留换行格式、BOM、注释和未结构化源码。
- 在自由画布上摆放紧凑节点。节点位置不会改变程序执行顺序。
- 拖动画布、平移缩放、框选和对齐节点，并通过 minimap 浏览较大的流程图。
- 从兼容端口的任一端开始连线，重新接线，或把兼容积木插入现有连线。
- 只有在生成合法 C、完整重解析并验证目标 CFG 后，连线才会提交。
- `raw`、宏和 partial CFG 区域仍可查看和运行，但不允许危险拓扑编辑。

### 运行、追踪和分析真实行为

- 使用本机 Apple clang 编译运行程序。
- 查看映射到源码和积木的编译诊断，并运行 ASan/UBSan 或有界 `leaks`
  检查。
- 通过临时影子源码追踪实际输入触发的路径；Trace 不会修改项目源码。
- 分别查看编译耗时、墙钟时间、峰值内存、峰值进程数、输出大小、执行节点、
  操作次数和终止原因。
- 对多个输入规模重复运行并比较中位数，不生成没有依据的综合效率分数。
- 明确区分真实运行和教学模拟。模拟路径不会写入真实性能历史，也不会冒充
  真实输出。

### 通过真实任务完成第一个算法

新版第一课会创建独立的“扫描求最大值”沙箱，并带你完成：

1. 运行正确程序并核对输出；
2. 追踪比较条件的 true 和 false 分支；
3. 插入缺失的“更新最大值”积木；
4. 阅读 Trace 和 Benchmark 图表；
5. 复现并修复错误比较符；
6. 把最大值扫描迁移为最小值扫描。

课程只有在工作区、源码、案例、运行和 Trace 证据全部匹配后才能推进。它不
使用人物对白，也不能通过连续点击“下一步”伪造完成状态。

### 把预设和 Library 当作工作工具

- 使用 80 个版本化预设，覆盖 C 基础、I/O、数组、字符串、指针、内存、
  数据结构、算法模式和虚拟流程控制。
- 在 114 个 Library 词条中搜索关键词、别名、语法、示例、复杂度、常见
  错误和相关概念。
- 普通学习搜索不再显示 renderer、路径 ID 和 revision 等内部实现术语。
- 从同一 Library 界面进入软件手册、小教程、诊断说明和扩展文档。

### 更直接、更紧凑的交互

- 使用纯文字顶部 Dock 和唯一的主要运行操作。
- 在 Dashboard 中单击整行直接打开项目。
- 独立调整主要面板尺寸，拖动工具窗口，并防止窗口移出可用区域。
- 单击节点选择，移动后开始拖拽；双击或按 Enter 打开详情。
- 使用 Quick Open 和键盘导航访问文件、面板、设置、Library 和命令。
- 在中文和英文间切换。首次启动遵循 macOS 首选语言；背景和主题会同步
  更新工作台与弹窗。

## 可选 AI 助手

AI 助手支持用户自备密钥连接 OpenAI、Anthropic、Gemini、OpenRouter、
DeepSeek、智谱 GLM、Kimi 中国区和 Kimi 国际区。

联网 AI 默认不启用。API 密钥通过 Electron `safeStorage` 加密保存，
renderer 无法读取明文或密文。主进程只会连接用户所选厂商的官方 HTTPS
主机，不会拿同一个密钥轮询多个服务。

默认请求包含用户问题、当前函数、有界诊断和控制流摘要、有界运行证据，
以及有限的最近对话。它不包含文件路径、stdin、程序参数、fixture 或凭据。
只读模式不发送完整源码。用户显式切换到“建议修改”或“代理”后，该模式下
每次请求会包含完整 `main.c`；AI 弹窗会持续显示这一外发提示。

每个托管工作区拥有一个有容量上限的本地 AI Project，可保存多批对话。
厂商凭据仍保存在独立的全局加密存储中。

AI 修改源码的权限默认关闭。用户显式启用复核或受控执行后，模型只能返回
结构化替换提案。提案仍须匹配当前工作区 revision 和源码指纹，生成精确
diff 和检查点，完成重解析、无损往返与 CFG 安全验证。过期或不安全提案
不会修改 `main.c`。

启用网络 AI 前，请阅读 [Privacy](../../PRIVACY.md) 和
[Security Policy](../../SECURITY.md)。

## v0.0.1 包含的开发历程

### M0：安全桌面基础

- 建立 Electron main、preload 和 renderer 边界。
- 加入具名 IPC、有界请求、本机编译执行、进程组清理、资源限制和
  trusted-only 授权。
- 建立课程 C 金样本和故障样本回归。

### M1：无损 C 投影

- 建立源码文档与积木投影，保持逐字符往返。
- 加入注释归属、解析恢复、金样本和 fuzz 覆盖。

### M2：交互式算法面板

- 加入语句级积木、代码/积木双向高亮、变量声明和使用定位、确定性解释与
  运行面板。

### M3：受控编辑

- 加入字面量和运算符修改、条件和循环表单、语句插删、保守重命名、精确
  diff 确认，以及源码权威的撤销和重做。

### M4：本地组装工作区

- 加入 Documents 托管 Dashboard、自动保存、可拖拽预设、自定义积木
  生命周期、扩展点和工业化组装布局。

### M5：证据与诊断

- 加入结构化 CFG、def-use、到达定义、循环事实、确定性数组越界 finding，
  以及保守的直接唯一堆句柄内存 typestate。
- 保留 M0–M5 金样本、等价性、架构、fuzz 和 Electron 回归门禁。

### 历史 Beta 快照

- `v0.1.0-beta.1` 加入自由流程工作台、源码安全连线、影子 Trace、运行
  历史、80 个预设、114 个 Library 词条、sidecar、CI 和 Universal DMG
  发布基础。
- `v0.1.0-beta.2` 扩展受验证 Apple clang 范围，并强化 SDK 路径校验。
- `v0.1.0-beta.3–10` 集中强化 `leaks` 有界监督、进程清理、positive
  control、诊断证据和托管 macOS 环境兼容。
- `v0.1.0-beta.5–9` 同时修复 Electron E2E 竞态和 CRLF 编辑、撤销同步。
- `v0.1.0-beta.11` 调整确定性 5,000 轮 fuzz 的超时预算，不改变种子、
  输入或不变量。
- `v0.1.0-beta.12` 只在 helper 完成清理并取得严格数值泄漏证据后，接受
  文档规定的 `leaks --atExit` leader 结果。

当前版本在这些快照之上加入 v6 教程、分析图、多厂商 AI、项目级对话、
受控 AI 改码、完整双语界面、Quick Open、统一主操作、电线式连线和整体
交互清理，并修复了源码编辑后新 CFG 被旧源码指纹拒绝、分析长期停留在
等待状态的问题。

## 迁移说明

### 现有项目

现有托管项目会原地打开。迁移过程不会改写 `main.c`。

- `flow-view.json` v1 会按需迁移为基于锚点的 v2。如果重复源码使某个
  锚点不唯一，只会丢弃对应的位置、选择或详情状态。
- `scenarios.json`、`run-history.json`、`tutorial-progress.json` 和
  `ai-project.json` 只在使用对应功能时创建。
- 不支持、过期或损坏的 sidecar 会被忽略或重置，不会修改源码。
- 自定义预设被弃用或退休后，已有项目仍保留模板版本和源码快照。

### AI 配置

旧 AI Provider 配置会进入“需要重新连接”状态。重新输入密钥，并选择厂商
和模型。这一步把新的加密凭据包绑定到单一厂商，避免猜测旧密文属于哪个
服务。

旧版本会忽略新增的 `ai-project.json`。降级后这些对话不可见，但旧版本
仍可打开 `main.c` 和其支持的其他项目数据。

### 版本线重置

`v0.0.1` 的 SemVer 优先级低于历史 `v0.1.0-beta.1–12` 标签。自动版本
比较可能把公开版判断为更旧。请显式安装 `v0.0.1`，后续更新以重置后的
公开版本线为准。

## 系统要求

- macOS；
- Universal DMG 支持的 Apple Silicon 或 Intel Mac；
- Apple clang 17.x–21.x，用于本地编译、运行、Trace 和诊断。

联网 AI 还需要所选厂商的账户、API 密钥和网络连接。其余工作台功能保持
本地运行。

## 安装与校验

`v0.0.1` GitHub Release 提供一个 Universal DMG 和
`SHA256SUMS.txt`。DMG **未签名、未公证**。

1. 从同一个 Release 下载两个文件。
2. 在下载目录运行：

   ```sh
   shasum -a 256 --check SHA256SUMS.txt
   ```

3. 校验失败时立即停止。
4. 挂载 DMG，把应用复制到 **Applications**。
5. 如果 Gatekeeper 阻止首次启动，在 Finder 中按住 Control 单击应用，
   选择 **打开** 并确认。
6. 如果仍被阻止，打开 **系统设置 → 隐私与安全性**，为这个应用选择
   **仍要打开**。

不要全局关闭 Gatekeeper。

## 已知限制

- DMG 未签名、未公证。
- 当前只面向 macOS 和本机 Apple clang 工具链。
- 任意 C 可以导入和运行，但只有经过验证的结构子集可以在画布上改线。
- `raw`、宏、partial CFG 和歧义区域会主动拒绝危险编辑。
- Trace 证明观察到的路径事件，不提供任意变量值采样。
- 运行指标只描述当前机器、工具链、源码和输入，不能独立证明算法复杂度。
- AI 回答来自所选第三方服务，可能错误，不能替代编译器、测试、Trace 或
  源码验证。

发布候选锁文件已于 2026-07-14 运行 `npm audit --audit-level=high`，当时未
报告已知漏洞；这只是该时间点的数据库结果，不代表未来不会出现新公告。
审计摘要见 [`npm-audit-v0.0.1.json`](../security/npm-audit-v0.0.1.json)。
