# AlgoLatch v0.1.1-preview.1 for macOS and Windows

> **Testing preview:** this release substantially expands the tutorial system,
> animation and runtime-state visualization. Some courses may still contain
> interaction, layout, localization or result bugs. If you find one, open a
> [GitHub issue](https://github.com/Ice-teapop/algolatch/issues/new/choose) and
> include the course number, operating system and display scale, reproduction
> steps, and a screenshot. Do not post private source code, API keys or
> exploitable security details in a public issue.

This prerelease provides two packages built and verified from the same source
revision:

- `AlgoLatch-0.1.1-preview.1-unsigned-universal.dmg` for Apple Silicon and
  Intel Macs;
- `AlgoLatch-Setup-0.1.1-preview.1-unsigned-x64.exe` for Windows 10/11 x64.

## What is included

- A standalone FOA tutorial module with a validated 120-course catalog and
  searchable Library knowledge projections.
- Course-specific semantic scenes for values, arrays, branches, stacks,
  pointers, matrices and later transition exercises instead of code highlight
  alone.
- Interactive input paths and learner actions where the course concept calls
  for them, with source highlighting synchronized to the current semantic step.
- Stable tutorial timelines with play, pause, step, seek and speed controls,
  plus reduced-motion behavior shared by macOS and Windows.
- Runtime-evidence contracts that distinguish real Trace facts, structural
  matching and teaching simulation; simulated values do not enter real run
  history.
- Responsive course layouts and independently scrollable regions for smaller
  windows and display scaling.
- Refactored tutorial, scene, runtime and style modules to keep the expanded
  curriculum maintainable.

## Known preview limitations

- The new course surface is broad and remains under active testing. Individual
  courses can still have animation timing, text overlap, responsive layout,
  translation or task-validation defects.
- Some course scenes intentionally use bounded teaching models because the
  current Trace boundary proves executed lines and branches but does not sample
  arbitrary C variables.
- The workbench remains a single-`main.c` application. Macros, `goto`, parser
  recovery and partial CFG regions can reduce structured editing capability.
- The macOS application is ad-hoc signed, not Developer ID signed or notarized.
  The Windows installer is not Authenticode-signed. Both operating systems may
  display an untrusted-publisher warning.

## Installation and trust boundary

Download only from this GitHub Release and verify `SHA256SUMS.txt`.

### macOS

1. Download the Universal DMG and `SHA256SUMS.txt`.
2. Run `shasum -a 256 --check SHA256SUMS.txt` in the download directory.
3. Open the DMG and copy **AlgoLatch** to **Applications**.
4. If Gatekeeper blocks this unsigned preview, Control-click the app, choose
   **Open**, and approve it once. Do not disable Gatekeeper globally.

### Windows

1. Download the x64 EXE and `SHA256SUMS.txt`.
2. Verify the installer with `Get-FileHash` before opening it.
3. Run the one-click per-user installer. No administrator access, external
   compiler, gateway script or `PATH` change is required.

Replacing or uninstalling the application does not remove managed projects in
Documents. API credentials and application state remain in the operating
system user profile.

## Verification performed by the release workflow

- Release-boundary, license notices, formatting, type and dependency checks.
- Unit, integration and architecture tests for the source-authoritative
  workbench, tutorial catalog, semantic scenes, Trace and runtime evidence.
- Stabilized the bilingual Library search coverage and asynchronous Trace
  cancellation gates so macOS runner scheduling load cannot cause false failures.
- Fixed the binary-search course marker lane at compact widths and browser zoom
  levels so `low`, `mid` and `high−1` remain readable without visual overlap.
- Targeted Electron regressions for the flow canvas and native AI window.
- macOS Universal build, ad-hoc signature validation, DMG install, launch and
  application probe.
- Windows locked-toolchain verification, real install, launch, native C
  compile/run, uninstall and project-retention checks.
- Independent platform manifests followed by one public SHA-256 manifest over
  the final DMG and EXE.
